start here: the three letters

linux permissions are built from just three letters. a dash - in a slot means that permission is switched off.

r= 4
read — view a file's contents (or list what's inside a folder)
w= 2
write — change or delete a file (or add/remove files in a folder)
x= 1
execute — run a file as a program (or enter/cd into a folder)

add up the values in each group to get its digit (e.g. rw- = 4+2 = 6). these three letters repeat three times — once for the owner, once for the group, once for others.

the formula

each group of three letters adds up to one digit. three groups → a three-digit number.

rwx = 4+2+1 = 7
rw- = 4+2    = 6
r-x = 4   +1 = 5
r-- = 4      = 4
so rwxr-xr-x → 7 5 5 → 755

the 777 trap
777 = rwxrwxrwx: everyone can read, write, and execute.

It's almost never the correct fix. If something only works at 777, you usually have an ownership or group problem — fix that instead of opening the file to the whole world. Reach for the narrowest permission that works (often 755 or 775).
permissions vs ownership
chmod changes what the owner / group / others may do.
chown changes who the owner and group are.

-rw-r-----  user  www-data  config.php
Here user (owner) can read & write, the www-data group can read, and others get nothing. To let the group edit it, change permissions (chmod g+w). To make a different user the owner, change ownership (chown). A lot of "permission" problems are really ownership problems.
symlinks
A symlink almost always shows as lrwxrwxrwx, and those bits are ignored. chmod on a symlink normally changes its target, not the link itself — so don't reason about a link's own rwx bits.
not covered yet
Beyond the basics there are three special bits that appear in the permission string: setuid (s in the owner's x spot), setgid (s in the group's x spot), and the sticky bit (t in others' x spot) — e.g. drwxrwxrwt on /tmp. They add a fourth leading octal digit and aren't part of this trainer yet.
